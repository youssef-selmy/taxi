Passkey Authentication Setup ​

Passkey authentication (WebAuthn) provides a secure, passwordless login experience using biometric authentication like Face ID, Touch ID, or Windows Hello. This guide covers how to enable and configure passkey support for your BetterTaxi deployment.

🔑 What is Passkey Authentication? ​

Passkeys are a modern authentication method that replaces passwords with cryptographic key pairs stored securely on the user's device. Benefits include:

  • Phishing-resistant - Credentials are bound to the origin domain
  • No password to remember - Uses biometrics (fingerprint, face) or device PIN
  • Cross-device sync - Apple and Google sync passkeys across devices
  • Fast login - One-tap authentication after initial setup

⚙️ Backend Configuration ​

Configure the following environment variables in your backend API servers (rider-api, driver-api):

env
# Passkey / WebAuthn Configuration
PASSKEY_RP_ID=yourdomain.com
PASSKEY_RP_NAME=YourAppName
PASSKEY_ORIGIN=https://yourdomain.com
VariableDescriptionExample
PASSKEY_RP_IDThe domain identifier (Relying Party ID). Must match the domain users access.api.bettersuite.io or localhost
PASSKEY_RP_NAMEHuman-readable app name shown during registrationBetterTaxi, MyRideApp
PASSKEY_ORIGINFull origin URL including protocolhttps://app.bettersuite.io

WARNING

The PASSKEY_RP_ID must exactly match the domain where users access your app (without protocol). For development, use localhost.

Default Values (if not set):

  • PASSKEY_RP_ID: localhost
  • PASSKEY_RP_NAME: BetterSuite
  • PASSKEY_ORIGIN: https://localhost

🗄️ Database Requirements ​

The passkey feature requires the auth_method table. Ensure the migration has been applied:

bash
# Migration file: 1763232553000-auth-method-module.ts
# This runs automatically on API startup

If you're upgrading from a previous version, restart your API servers to apply pending migrations.

📱 Frontend Configuration ​

Enable passkey authentication in your Flutter apps by setting the environment variable in .env.dev and .env.prod:

env
# Enable passkey authentication
PASSKEY_ENABLED=true

Location of .env files:

  • Rider App: apps/taxi-rider-frontend/.env.dev / .env.prod
  • Driver App: apps/taxi-driver-frontend/.env.dev / .env.prod

TIP

After changing environment variables, rebuild the app to apply changes.

🍎 iOS Setup (Required) ​

iOS requires Associated Domains to be configured for passkey authentication to work. This links your app with your server domain.

Step 1: Configure environment variables

Add these to your root .env file:

env
# Apple App Site Association (for passkey)
APPLE_TEAM_ID=YOUR_TEAM_ID
RIDER_BUNDLE_ID=com.yourcompany.rider
DRIVER_BUNDLE_ID=com.yourcompany.driver

TIP

Find your Team ID in the Apple Developer Portal under Membership Details.

Step 2: Deploy and verify the association file

Both rider-api and driver-api automatically serve the apple-app-site-association file at:

  • https://yourdomain.com/.well-known/apple-app-site-association

After deploying, verify the file is accessible:

bash
curl https://yourdomain.com/.well-known/apple-app-site-association

Expected response:

json
{
  "webcredentials": {
    "apps": [
      "TEAMID.com.yourcompany.rider",
      "TEAMID.com.yourcompany.driver"
    ]
  }
}

Step 3: Update iOS entitlements

The Flutter apps are pre-configured with Associated Domains. Update the domain in:

  • apps/taxi-rider-frontend/ios/Runner/Runner.entitlements
  • apps/taxi-driver-frontend/ios/Runner/Runner.entitlements

Replace yourdomain.com with your actual PASSKEY_RP_ID domain:

xml
<key>com.apple.developer.associated-domains</key>
<array>
    <string>webcredentials:yourdomain.com</string>
</array>

WARNING

For local development, add ?mode=developer to bypass server verification:

xml
<string>webcredentials:yourdomain.com?mode=developer</string>

Remove ?mode=developer for production builds!

Step 4: Enable Associated Domains in Apple Developer Portal

  1. Go to Apple Developer Portal
  2. Navigate to Certificates, Identifiers & Profiles
  3. Select your App ID (for both rider and driver apps)
  4. Enable Associated Domains capability
  5. Rebuild and re-sign your app

Important

The domain in your iOS entitlements must exactly match the PASSKEY_RP_ID configured on your backend. Mismatched domains will cause passkey registration/login to fail.

🚀 User Flows ​

Registration Flow (Post-Login):

  1. User logs in with phone/email
  2. After successful login, app prompts to set up passkey
  3. User authenticates with biometrics
  4. Passkey is registered and linked to account

Login Flow:

  1. User taps "Login with Passkey" on the login screen
  2. Device shows biometric prompt
  3. User authenticates with Face ID/Touch ID/PIN
  4. App receives JWT token and logs user in

🔧 Troubleshooting ​

❌ "Passkey not supported on this device"

Cause:

  • Device doesn't meet minimum OS requirements
  • Google Play Services not available (Android)
  • Running on web/desktop platform

Resolution:

  • Ensure iOS 16+ or Android 9+ with Play Services
  • Use phone/email login as fallback

❌ "Failed to begin passkey authentication"

Cause:

  • Backend API not reachable
  • Redis not running (challenges stored in Redis)
  • Invalid PASSKEY_RP_ID configuration

Resolution:

  • Verify API is running and accessible
  • Check Redis connection
  • Ensure PASSKEY_RP_ID matches your domain

❌ Passkey works in dev but not production

Cause:

  • PASSKEY_RP_ID or PASSKEY_ORIGIN mismatch with production domain

Resolution:

  • Update environment variables to match production domain exactly
  • PASSKEY_RP_ID should be the domain without protocol (e.g., app.example.com)
  • PASSKEY_ORIGIN should include the protocol (e.g., https://app.example.com)

Backend:

  • [ ] Set PASSKEY_RP_ID to your domain
  • [ ] Set PASSKEY_RP_NAME to your app name
  • [ ] Set PASSKEY_ORIGIN to your full origin URL
  • [ ] Set APPLE_TEAM_ID to your Apple Team ID
  • [ ] Set RIDER_BUNDLE_ID and DRIVER_BUNDLE_ID to your app bundle IDs
  • [ ] Ensure Redis is running
  • [ ] Restart API servers to apply migrations

Frontend:

  • [ ] Set PASSKEY_ENABLED=true in .env files
  • [ ] Rebuild Flutter apps after config changes

iOS (Required):

  • [ ] Update Runner.entitlements with your domain (rider and driver apps)
  • [ ] Enable Associated Domains in Apple Developer Portal
  • [ ] Verify apple-app-site-association is accessible at your domain
  • [ ] Remove ?mode=developer suffix for production builds
